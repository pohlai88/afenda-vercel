## Overview

**Structured query (demo)** is Lynx's **Canonical intake** layer preview: ask in plain English, receive **read-only SQL** against a governed demo table, see results in a table or chart, and optionally explain the SQL.

This is **not** production analytics over live ERP tables.

## Before you start

- ERP permission **`lynx.workspace.search`** to reach the Lynx page (demo Server Actions use `requireOrgSession()` once you are on the page).
- **`FLAG_LYNX_STRUCTURED_QUERY_DEMO_ENABLED`** — when off, Lynx shows a policy notice instead of the demo form.
- Seeded rows in **`lynx_demo_unicorn`** for your organisation — empty tables return empty results.

<Callout type="warn" title="Demo only">
  This layer queries **`lynx_demo_unicorn` only** — not contacts, HRM, or Orbit tables. It validates NL→SQL guardrails before production intake ships.
</Callout>

## How to use the demo

<Steps>
  <Step>
    Open **Lynx** at `/{locale}/o/{orgSlug}/dashboard/lynx` → **Structured query (demo)**.
  </Step>
  <Step>
    Enter a question — for example: *Average valuation by industry in the United States*.
  </Step>
  <Step>
    Click **Run query**. Lynx generates SQL, validates it, executes against Postgres, and returns rows (cap **200** — `LYNX_NL_DEMO_ROW_CAP`).
  </Step>
  <Step>
    Review **Generated SQL**, switch between **Table** and **Chart** tabs, or click **Explain SQL** for a prose walkthrough.
  </Step>
</Steps>

## Safety guardrails

Generated SQL must pass static validation:

| Rule | Reason |
| --- | --- |
| `SELECT` only | No mutations via NL→SQL |
| `FROM` must be exactly `"lynx_demo_unicorn"` | Prevents cross-table access |
| `LIMIT` required and capped at 200 | Protects database and UI |
| No DDL/DML keywords | Blocks destructive statements |

Failed validation returns an expected error — not a silent fallback query.

## Audit

Successful executions emit **`erp.lynx.nl_demo.query`** with resource type `lynx.nl_demo`.

## When to use something else

| Need | Use instead |
| --- | --- |
| Policy answers from documents | [Truth retrieval](/ask-docs/lynx/truth-retrieval) |
| Live Orbit or contact facts | [Operator assist](/ask-docs/lynx/operator-assist) |
| Production ERP reporting | Governed dashboards and modules — not this demo |

## Troubleshooting

<Accordions type="single">
  <Accordion title="Demo section shows policy notice">
    `FLAG_LYNX_STRUCTURED_QUERY_DEMO_ENABLED` is off — enable via environment policy or use truth/operator layers instead.
  </Accordion>
  <Accordion title="Query returns zero rows">
    Confirm demo seed data exists for your `organizationId` in `lynx_demo_unicorn`.
  </Accordion>
  <Accordion title="Validation rejected my generated SQL">
    Rephrase to require a single-table `SELECT` with an explicit `LIMIT` — cross-table or DML patterns are blocked by design.
  </Accordion>
  <Accordion title="Chart tab unavailable">
    Result shape may not support visualization — try a question that yields numeric aggregates by category.
  </Accordion>
</Accordions>
