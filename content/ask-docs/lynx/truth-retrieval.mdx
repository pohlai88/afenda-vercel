## Overview

**Truth retrieval** is Lynx's evidence-first Q&A layer. You ask a question; Lynx embeds the query, retrieves the top matching **knowledge chunks** for your organisation, and streams an answer constrained to that evidence.

Use truth retrieval when you need: *what does our written evidence say about this?*

## Before you start

- ERP permission **`lynx.workspace.search`** for the Lynx page and `POST /api/erp/lynx/truth-search`.
- At least one **knowledge chunk** with a successful embedding — see [Knowledge substrate](/ask-docs/lynx/knowledge-substrate).
- AI Gateway auth for embeddings and generation.

<Callout type="info" title="Default retrieval depth">
  Similarity search returns the top **8** passages by default (`LYNX_TRUTH_TOP_K`). Thin corpora produce explicit limitation text instead of invented policy.
</Callout>

## How truth retrieval works

<Steps>
  <Step>
    Enter a natural-language **question** (for example: "Why was this return rejected?").
  </Step>
  <Step>
    Lynx embeds the question and runs similarity search over `knowledge_chunk` rows scoped to your `organizationId`.
  </Step>
  <Step>
    The model receives a system prompt built from retrieved passages and streams markdown over **`POST /api/erp/lynx/truth-search`** (NDJSON).
  </Step>
  <Step>
    The UI parses the stream into **Answer**, **Evidence used**, **Limitations**, and **Next safe action** via `parseLynxTruthMarkdown`.
  </Step>
</Steps>

## Response sections

### Answer

Synthesis from retrieved passages only. If no chunks match, the answer states that coverage is insufficient rather than inventing policy.

### Evidence used

Each passage shows chunk **title**, excerpt, and **cosine distance** (lower = closer match). Evidence is listed from server retrieval metadata — not parsed from model prose alone.

### Limitations

When retrieval is thin, Lynx explains that answers are limited to stored chunks. Default guidance: add knowledge chunks for broader coverage.

### Next safe action

Actionable, policy-safe guidance — for example open the Lynx console to add evidence, or escalate to an owner. This is not an automatic ERP mutation.

## Audit

Successful queries emit **`erp.lynx.truth.query`** with a hashed query fingerprint — not the full question text in metadata when PII-sensitive.

## Troubleshooting

<Accordions type="single">
  <Accordion title="Empty evidence list">
    Add chunks on [Knowledge substrate](/ask-docs/lynx/knowledge-substrate) and confirm embeddings completed with gateway auth.
  </Accordion>
  <Accordion title="Stream error or disconnect">
    Check gateway credentials, model config (`LYNX_GENERATION_MODEL`), and network stability. Client shows a recovery message on malformed streams.
  </Accordion>
  <Accordion title="Answer invents policy not in evidence">
    Split chunks into tighter paragraphs and retry — truth prompts constrain the model to retrieved passages; empty retrieval should surface limitations instead.
  </Accordion>
  <Accordion title="I need live Orbit or contact counts">
    Use [Operator assist](/ask-docs/lynx/operator-assist) — truth retrieval reads written chunks, not live ERP tables.
  </Accordion>
</Accordions>
