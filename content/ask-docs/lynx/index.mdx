## Overview

**Lynx** is Afenda's **machine layer** — not a generic chatbot or copilot. It resolves truth from organisational evidence, streams governed recommendations, and exposes read-only tools where policy allows. Product copy uses **Lynx** and the eyebrow **The Machine**.

Lynx composes the **Knowledge** substrate (`#features/knowledge`) for pgvector chunks; it does not replace ERP modules such as [Orbit](/ask-docs/orbit) or [HRM](/ask-docs/hrm).

## Before you start

- ERP permission **`lynx.workspace.search`** to open `/{locale}/o/{orgSlug}/dashboard/lynx` and use truth retrieval.
- ERP permission **`lynx.workspace.predict`** for **Operator assist** (`POST /api/erp/lynx/operator`).
- AI Gateway auth (`AI_GATEWAY_API_KEY` locally or Vercel OIDC in production) for embedding-backed flows.
- Feature flags may hide operator assist or the NL→SQL demo — contact your administrator if sections show policy notices.

<Callout type="info" title="Public vs ERP Lynx">
  **Ask Lynx** on `/{locale}/ask-docs` is a separate public stack — no org session, no tenant chunks. See [Ask Lynx on docs](/ask-docs/lynx/ask-docs-chat).
</Callout>

## Open the Lynx console

<Steps>
  <Step>
    Sign in and open Nexus at `/{locale}/o/{orgSlug}/nexus`.
  </Step>
  <Step>
    Select **Lynx** in the left rail (`/dashboard/lynx`).
  </Step>
  <Step>
    Use **Truth retrieval** for document-centric questions, **Operator assist** for live cross-module facts, and **Knowledge substrate** to add or debug chunks.
  </Step>
  <Step>
    When enabled, try **Structured query (demo)** for read-only NL→SQL against the demo table only.
  </Step>
</Steps>

## Four product layers

| Layer | Code slug | Status |
| --- | --- | --- |
| Truth retrieval | `truth` | **Live** — `POST /api/erp/lynx/truth-search` |
| Operating briefs | `briefs` | Contract reserved — no UI yet |
| Canonical intake | `structured` | NL→SQL **demo** when flag enabled |
| Decision operator | `operator` | **Live (preview)** — `POST /api/erp/lynx/operator` |

Canonical metadata: `lib/features/lynx/lynx.contract.ts` (`LYNX_LAYERS`, `LYNX_AUDIT_ACTIONS`).

## Troubleshooting

<Accordions type="single">
  <Accordion title="Lynx is missing from the rail">
    Grant `lynx.workspace.search` through ERP RBAC. Better Auth org role alone does not surface the module.
  </Accordion>
  <Accordion title="Operator assist shows a policy notice">
    `FLAG_LYNX_OPERATOR_ENABLED` may be off, or you lack `lynx.workspace.predict`. Orbit tools additionally require `FLAG_LYNX_OPERATOR_ORBIT_TOOLS_ENABLED`.
  </Accordion>
  <Accordion title="Truth retrieval returns empty evidence">
    Add chunks on the same page under [Knowledge substrate](/ask-docs/lynx/knowledge-substrate) and confirm gateway auth for embeddings.
  </Accordion>
</Accordions>
