## Overview

**Ask Lynx** is the AI chat on **`/{locale}/ask-docs`** ‚Äî the documentation site you are reading now. It answers questions about Afenda's product using **public ask-docs content** only.

## Before you start

- No Afenda sign-in required ‚Äî the ask-docs surface is excluded from the session cookie gate in `proxy.ts`.
- Local development needs `AI_GATEWAY_API_KEY` (or Vercel OIDC on deploy) for model calls.
- Production rate limiting expects Upstash Redis (`UPSTASH_REDIS_REST_*`) when configured.

<Callout type="info" title="Not ERP Lynx">
  This chat does **not** import `#features/lynx` and does **not** access your organisation's ERP data, knowledge chunks, or Orbit state. CI enforces this via `pnpm lint:public-lynx-contract`.
</Callout>

## ERP Lynx vs Ask Lynx

| | ERP Lynx | Ask Lynx (docs) |
| --- | --- | --- |
| **Surface** | `/{locale}/o/{orgSlug}/dashboard/lynx` + summon | `/{locale}/ask-docs` |
| **Auth** | `requireOrgSession()` | None ‚Äî public |
| **Data scope** | Org `knowledge_chunk` + governed tools | ask-docs MDX / search index |
| **Routes** | `app/api/erp/lynx/*` | `app/api/chat/route.ts` |
| **Audit** | `erp.lynx.*` | No IAM audit |
| **Imports** | `#features/lynx` | `lib/ask-docs/public-lynx*` + Vercel AI SDK |

Never route public chat through the ERP Lynx module ‚Äî that would create an IDOR risk and corrupt the audit contract.

## How to use Ask Lynx

<Steps>
  <Step>
    Browse to any ask-docs page (for example `/{locale}/ask-docs`).
  </Step>
  <Step>
    Click the round **Lynx** trigger wired in `app/(ask-docs)/[locale]/ask-docs/layout.tsx`.
  </Step>
  <Step>
    Ask a product question ‚Äî suggested prompts appear on first open.
  </Step>
  <Step>
    Read the streamed reply. Answers ground in indexed documentation via flexsearch and processed markdown ‚Äî not your tenant database.
  </Step>
</Steps>

## Knowledge sources

Ask Lynx retrieves from public docs infrastructure:

- Fumadocs **Orama** search at `/api/ask-docs-search` (locale query param)
- Processed markdown at `/llms.mdx/ask-docs/‚Ä¶`
- Corpus exports `/llms.txt` and `/llms-full.txt` where configured

POST bodies are parsed with **`readPublicLynxChatRequestBody`** ‚Äî not raw `req.json()` ‚Äî to enforce size caps.

## When to use which Lynx

| Question type | Use |
| --- | --- |
| How do I configure leave in Afenda? | **Ask Lynx** (this page) |
| What does our written return policy say? | **ERP truth retrieval** (signed in) |
| How many blocked Orbit items do we have? | **ERP operator assist** (signed in) |
| What is in our contact directory? | **ERP operator assist** ‚Äî not public chat |

## Troubleshooting

<Accordions type="single">
  <Accordion title="Ask Lynx returns an error locally">
    Set `AI_GATEWAY_API_KEY` in `.env.local` via `pnpm env:sync` or rely on Vercel OIDC in preview/production.
  </Accordion>
  <Accordion title="Answer cites outdated docs">
    Rebuild or redeploy so Fumadocs recompiles `content/ask-docs` ‚Äî static pages use processed MDX from `.source/`.
  </Accordion>
  <Accordion title="I need tenant-specific policy text">
    Sign in and use [Truth retrieval](/ask-docs/lynx/truth-retrieval) against your organisation's knowledge chunks.
  </Accordion>
</Accordions>
