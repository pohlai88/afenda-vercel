## Overview

**Ask Lynx** is the AI chat on **`/{locale}/ask-docs`** ‚Äî the documentation site you are reading now. It answers questions about Afenda's product using **public ask-docs content** only.

<Callout type="info" title="Not ERP Lynx">
  This chat does **not** import `#features/lynx` and does **not** access your organisation's ERP data, knowledge chunks, or Orbit state.
</Callout>

## ERP Lynx vs Ask Lynx

| | ERP Lynx | Ask Lynx (docs) |
|---|---|---|
| **Auth** | `requireOrgSession()` | None ‚Äî public |
| **Data scope** | Your org's chunks + governed tools | ask-docs MDX / search index |
| **Routes** | `app/api/erp/lynx/*` | `app/api/chat/route.ts` |
| **Audit** | `erp.lynx.*` | No IAM audit subject |
| **Import rule** | `#features/lynx` | Vercel AI SDK + `lib/ask-docs/public-lynx*` |

Never route public chat through the ERP Lynx module ‚Äî that would create an IDOR risk or corrupt the audit contract.

## How to use it

<Steps>
  <Step>
    Browse to any ask-docs page (for example `/en/ask-docs`).
  </Step>
  <Step>
    Click the round **Lynx** trigger (bottom of the layout).
  </Step>
  <Step>
    Ask a product question ‚Äî suggested prompts appear on first open.
  </Step>
  <Step>
    Read the streamed reply. Answers are grounded in indexed documentation, not your tenant database.
  </Step>
</Steps>

## Knowledge sources

Ask Lynx retrieves from public docs infrastructure:

- Fumadocs **Orama** search (`/api/ask-docs-search`)
- Processed markdown endpoints (`/llms.mdx/ask-docs/‚Ä¶`)
- Full corpus exports (`/llms-full.txt`) where configured

Locale-aware search supports configured ask-docs locales (for example `en`, `zh-CN`, `vi`, `ms`).

## Safety and abuse controls

Public endpoints have no org session, so defenses include:

- **Body size cap** via `readPublicLynxChatRequestBody` ‚Äî not raw `req.json()`
- **Rate limiting** when Upstash (or equivalent) is configured
- **CI contract** ‚Äî `pnpm lint:public-lynx-contract` blocks ERP Lynx imports in the public stack

## Branding

- UI label: **Ask Lynx** or **Lynx** ‚Äî consistent with ERP surfaces
- Avoid: chatbot, copilot, AI assistant
- Icons: `public/icons/lynx/Lynx-transfrom-*.png` via `LynxBrandMark`

## When to use which Lynx

| Question type | Use |
|---|---|
| How do I configure leave in Afenda? | **Ask Lynx** (this page) |
| What does our written return policy say? | **ERP truth retrieval** (signed in) |
| How many blocked Orbit items do we have? | **ERP operator assist** (signed in) |
| What is in our contact directory? | **ERP operator assist** ‚Äî not public chat |
