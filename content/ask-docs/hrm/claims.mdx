## Overview

**Claims** let employees submit reimbursable spend with typed line items and attachments. Approvers work a pending inbox; approved unpaid claims can feed the next payroll period so finance does not double-pay.

## Submission

Employees pick a claim type (currency-aware), describe the spend, and attach evidence stored in governed Blob storage. Drafts can be cancelled; submitted rows follow the approval machine.

## Approvals

Approvers see amount, policy hints, and evidence thumbnails. Denials return structured reasons so employees can resubmit with corrections.

<Callout type="info" title="Portal parity">
  The employee portal exposes claim submission for authenticated portal users — authority still resolves from portal access rows, not guessed client state.
</Callout>
