## Overview

**Onboarding** ties checklist templates to employment contracts. When someone joins, Afenda materialises **task instances** with due ordering, required flags, and completion evidence. **Offboarding** mirrors the discipline for exits — tasks close access paths instead of opening them.

## Before you start

- Maintain boarding **templates** before activating instances — the best-matching template wins per contract using deterministic scoring.
- Link policy acknowledgements and document uploads to tasks when compliance requires evidence.
- For exits, use the dedicated [Offboarding](/ask-docs/hrm/offboarding) workbench when managing clearance at scale.

## Activating a boarding checklist

<Steps>
  <Step>Open the employee record and locate the **Boarding** panel.</Step>
  <Step>Pick the template (or accept the system suggestion) before activation.</Step>
  <Step>Mark tasks complete as evidence arrives — partial saves are allowed.</Step>
  <Step>Resolve **blocked** instances when a task surfaces why work stopped.</Step>
</Steps>

## Templates

Templates encode default tasks (IT access, payroll profile, policy acknowledgements). The best-matching template wins per contract using deterministic scoring so behaviour is reproducible.

## Instance lifecycle

Instances move through explicit states (`pending`, `in_progress`, `blocked`, `completed`). Blocked rows surface why work stopped so HR can intervene without guessing.

<Callout type="info" title="Evidence discipline">
  Completion without evidence is allowed only where the template marks a task as optional — required tasks should attach vault uploads or acknowledgement records.
</Callout>

## Troubleshooting

<Accordions type="single">
  <Accordion title="No template suggested for a new hire">
    Confirm an active template matches the contract type and department scoring rules.
  </Accordion>
  <Accordion title="Task stays blocked">
    Open the task detail for the blocking reason — often a missing signature, document, or policy acknowledgement.
  </Accordion>
</Accordions>
