## Overview

**Onboarding** ties checklist templates to employment contracts. When someone joins, Afenda materialises **task instances** with due ordering, required flags, and completion evidence. **Offboarding** mirrors the discipline for exits — tasks close access paths instead of opening them.

## Templates

Templates encode default tasks (IT access, payroll profile, policy acknowledgements). The best-matching template wins per contract using deterministic scoring so behaviour is reproducible.

## Instance lifecycle

Instances move through explicit states (`pending`, `in_progress`, `blocked`, `completed`). Blocked rows surface why work stopped so HR can intervene without guessing.

<Steps>
  <Step>Open the employee record and locate the **Boarding** panel.</Step>
  <Step>Pick the template (or accept the system suggestion) before activation.</Step>
  <Step>Mark tasks complete as evidence arrives — partial saves are allowed.</Step>
</Steps>
