## Overview

**Policies** connect narrative rules (handbooks, codes of conduct) to workforce execution. Versions are immutable once published; acknowledgements capture *who saw which revision at what time* for disputes and inspections.

## Publishing

Authors draft revisions, publish when ready, and optionally require acknowledgement campaigns. Employees see only the active version for their population.

## Acknowledgements

Acknowledgements are explicit user actions — no silent auto-consent. Missing acknowledgements surface in manager queues where configured.

## Execution hooks

Downstream modules (leave, expenses, devices) can consult acknowledgement state to gate risky actions. Hooks stay declarative: business rules live in HRM Server Actions, not client guesswork.
